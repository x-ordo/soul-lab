# Toss Viral Notes (실전 UX)

## 왜 '링크를 먼저 복사'하나?
`contactsViral` / 공유 UI는 **링크 자동 주입이 보장되지 않을 수 있음**(모듈/정책/클라이언트 버전에 따라 다름).
그래서 실전에서는 아래 패턴이 안전합니다.

1) **초대 링크 생성**
2) **클립보드에 링크 복사**
3) **연락처 UI 오픈**
4) 유저가 메시지 입력칸에 **붙여넣기** 후 발송

이 패턴이면, 기능이 100% 동작하고 심사에서도 "사용자 주도"가 명확해집니다.

## 동의(제3자 제공) 처리
- **개인 분석**: 약관 동의만으로 이용 가능
- **친구 초대**: 제3자 정보 제공 동의 후에만 버튼 활성화

심사 리스크를 줄이면서도 바이럴은 유지됩니다.
